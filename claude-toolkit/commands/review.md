현재 변경사항을 시니어 엔지니어 관점에서 코드 리뷰해줘.

## 리뷰 대상

$ARGUMENTS

(인자가 없으면 `git diff`로 현재 변경사항 전체를 리뷰)

## 검토 항목

1. 버그 가능성
2. 보안 취약점
3. 성능 이슈
4. 코드 가독성

## 규칙

- 중요한 것만 지적 (nitpick 금지)
- 각 이슈에 심각도 표시: **높음** / **중간** / **낮음**
- 수정 방법도 함께 제안
- 잘된 부분도 한두 개 언급

## 출력 형식

```
### [심각도] 이슈 제목
- 파일: `파일경로:라인`
- 문제: 설명
- 제안: 수정 방법
```
